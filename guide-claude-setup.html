💡 What even is this?

Claude is your AI assistant; Claude Code is the small app on your laptop that runs all your Clinic Catalyst skills - the content engine, the follow-up writer, all of it. Your account is the brain, the API key is the power cord that plugs it into your system, and the install puts it on your machine. Usage is tiny - most clinics spend about $5-15 a month.

Before the day · two quick jobs
1

Create your Claude account

Sign up at claude.ai with your clinic email.

Sign up - use this link
2

Make your API key and save it

Go to the Anthropic Console and sign in with the same email as your Claude account. Then, in order:

1. Billing first: in the left menu open Billing and add a card. It is usage-based - you only pay for what the system uses (roughly $5-15 a month for a clinic). A key made before billing is set up can't run anything, so card first.
2. Create the key: open API Keys, click Create Key, and name it Clinic Catalyst.
3. Copy it now - you only see it once. Paste it into a note or your password manager. Closed the window without copying? No harm - delete that key and make a new one.

console.anthropic.com →
Treat it like a password. Anyone with the key can use your usage - keep it private, never post it anywhere.
In the room · we run the install together, first thing
3

On a Mac

Press Cmd+Space, type "Terminal", hit enter - it is just a plain text box, nothing to break. Paste this one line and press enter:

curl -fsSL https://clinic-catalyst-au.github.io/cc-onboard/clinic-bootstrap.sh | bash

That single command sets up Claude Code, all the helper tools and your Clinic Catalyst skills automatically (~25-45 min, mostly downloads - it churns in the background while the workshop runs).

4

On Windows

Windows needs more pieces than Mac - Python, ffmpeg, Node, Git - and this one command installs ALL of them, then Claude Code, then your skills. Press the Start button, type "PowerShell", and open Windows PowerShell - the blue window. (Not Command Prompt, not Git Bash - the blue one.) Then paste this line and press enter:

irm https://clinic-catalyst-au.github.io/cc-onboard/clinic-bootstrap.ps1 | iex

That single line sets up: Git, Node, Python, ffmpeg (the video engine), Claude Code itself, the Python packages your skills use, the full Clinic Catalyst skill pack and your ~/Clinic folder. Same ~25-45 minutes as Mac, mostly downloads. No account sign-ups needed for any of this - and no GitHub account, ever.

What you'll see while it runs: lines of text scrolling - that's normal. Windows may pop up "Do you want to allow this app to make changes?" once or twice - click Yes. Keep the window open until it says DONE. The installer ends with a self-check that lists every piece as "ok" or tells you exactly what to flag with us.

If it says "winget not found": install "App Installer" from the Microsoft Store (aka.ms/getwinget), then paste the command again. That's the most common speed bump - and we're on your screen with you anyway.
5

Close the window, open a fresh one, prove it

This step matters: the claude command only exists in windows opened AFTER the install. So close Terminal or PowerShell completely, open a brand new one the same way as before, and type:

claude --version

A version number means everything worked. That's your proof.

Says "not recognized" or "command not found"? Don't panic - one paste fixes it for this window.
Windows: $env:Path += ";$env:USERPROFILE\.local\bin"
Mac: source ~/.zprofile
Then try claude --version again. Still stuck? That's what we're in the room for.
6

Start it, sign in, plug in your key

In that same window type claude and press enter. First run asks you to sign in - choose your Claude account (the one from step 1), and paste the API key you saved when it asks. Done - your skills are on your machine.

claude
Anything look wrong at any point? Copy the last few lines from your window and paste them into the Install Chat - we watch it live in the room and fix as we go.
