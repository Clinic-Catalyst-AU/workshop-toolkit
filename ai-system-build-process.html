Implementation Process

Build your AI system, step by step.

The exact, in-order process to stand up an AI agent across web chat, social, email, SMS and voice - with booking, confirmations, automations and Google presence. Every gotcha flagged so you don't lose the hours we did.

The golden rules (read first)

  • Do the slow, approval-based steps FIRST (phone bundle, DNS) - they take 1-3 business days.
  • Knowledge Base = what the bot KNOWS. Instructions = how it BEHAVES. Behaviour rules go in Instructions, never the knowledge base.
  • Trial in Suggestive mode (bot drafts, you approve), then flip to Auto-pilot.
  • Verify with a real test through each channel - never assume, and never trust one lucky pass.
0

Pre-work: phone number + regulatory bundle

We do this together on the day

Australian numbers need a one-time regulatory bundle. The only thing you bring is your business registration PDF (free download from ABN Lookup) - we set the bundle up together, screen by screen, on Day 1, so it's approved in time to build on Day 2. Not a task you do alone.

Bring this (PDF/JPG/PNG, under 5MB) - we'll sort the rest together on the day

  • Business registration (ASIC/ABN)
  • Proof of physical address - utility bill or lease in the business name (no PO Box)
  • Authorised rep photo ID
  • Proof of authorisation
  • Website / social link
Settings → Phone System → Regulatory Bundles

Create Bundle → Australia / Local / Business (individual not supported). Submit, wait for approval, then link it to the number (Phone Numbers → ··· → Select AU Regulatory Bundle to Link) and confirm the green "KYC Linked" tag.

Gotcha - the empty dropdownThe bundle asks for an "Address sid" from a dropdown that's empty until an Address exists. Create the Address FIRST (Regulatory Compliance → Addresses → Create Address) or you'll be stuck.
Gotcha - no AU portingYou can't port an Australian number into GHL (port-in is US-only). Buy a fresh AU number once the bundle is approved.
1

Email foundation: sending domain

Optional - skip for now

Not required to go live. GHL sends email fine on its own from day one. This step only makes emails come from your own domain for slightly better inbox placement - a nice-to-have you can add later. Skip it for the workshop.

Later, if you want it: Settings → Email Services → Dedicated Domain

Add your root domain so the From address shows your real address. GHL generates DNS records (DKIM, return-path CNAME, tracking CNAME, Mailgun MX) to add at your DNS host.

Gotcha - SPFOnly ONE v=spf1 record is allowed per domain. If you already have one, MERGE GHL's includes into it - never add a second SPF record, or all email breaks.
Gotcha - DNS panelsSome panels won't let you delete records - edit the value instead. Verify with dig @your-nameserver; public resolvers cache stale data up to the record's TTL.
2

The AI agent (text)

Three content layers

  1. FAQ / Q&A - question + answer pairs, pasted into the Q&A trainer.
  2. Knowledge Base - one document (.docx) uploaded as the deeper reference.
  3. Instructions / Personality / Goals - the behaviour: tone, the funnel/routing, pricing rules, and the booking-capture rule.
Gotcha - KB vs InstructionsBehaviour (collecting details, routing, what to quote) must live in Instructions, not the knowledge base - or the bot treats it as trivia and ignores it.

Voice + funnel to bake in

  • Plain language, warm, no jargon.
  • The funnel: no idea where to start → the low-cost starter; wants it built → book a call.
  • Pricing: only quote the one fixed low-ticket price; everything bespoke routes to the call.
Gotcha - stale snapshot contentIf cloned from a snapshot, the Q&A, knowledge base and workflows carry OLD content (wrong prices, dead products). Audit and align ALL of it before going live.
3

Booking + confirmations

Capture details before booking

Gotcha - blank bookingsOn Instagram/Facebook, GHL only gets the person's name, never email/phone. The bot MUST be instructed to ask for and confirm name, email + mobile (with country code) BEFORE booking - or it books a blank contact and nothing can be sent.

Calendar

  • Assign a host to the calendar (Zoom links generate off the host's connected Zoom).
  • Set Meeting Location → Zoom so the confirmation contains a link.
Gotcha - the confirmation that actually worksGHL's built-in calendar confirmation only fires for bookings made through the calendar/widget. Bot-created bookings do NOT trigger it. Build a WORKFLOW with a "Customer Booked Appointment" trigger → Send Email + SMS (fires regardless of source). And remember: workflows ship as Draft and do nothing until you flip them to Publish.
4

Channels + go live

  1. Connect web chat, Facebook, Instagram, email. (TikTok is NOT a supported channel - funnel its traffic to link-in-bio/IG.)
  2. Enable the bot per channel (connecting is not the same as enabling).
  3. Test from a different account (your own DM won't trigger it) in Suggestive mode.
  4. Once replies look right, switch to Auto-pilot.
5

Voice agent

Configure the voice prompt with the same identity, funnel and no-jargon rules, plus the booking-capture rule (name/email/phone with country code before booking). For the voice itself, select the built-in Australian voice (Sophia) right inside GHL - no external voice account or extra subscription needed. If asked "are you AI?", answer honestly and warmly. Attach the approved number from Phase 0.

6

Inbound email into the system

Optional - big decision

To make cold emails land in the system (so the bot answers them), point the domain's MX to GHL/Mailgun as primary.

Gotcha - this moves the WHOLE inboxIt routes every address on that domain into GHL. Only do it if that address is dedicated to the system. The old MX's TTL (often 4 hours) means it takes hours to cut over - do it the day before, not the day of.
7

Website chat widget

  1. Sites → Chat Widget → connect it to the bot → enable the Web channel.
  2. Paste the embed <script> into the site HTML before </body>.
Gotcha - Content-Security-PolicyIf the site has a CSP, add the widget domains or it loads but can't talk to GHL: widgets.leadconnectorhq.com (script + frame), services.leadconnectorhq.com, backend.leadconnectorhq.com and wss://backend.leadconnectorhq.com (connect-src). Hard-refresh after deploy.
8

Automations (response + marketing)

Most ship as drafts in the snapshot - align the copy, then publish:

  • Speed-to-lead - instant SMS + email when a lead arrives
  • Missed-call-text-back - auto-SMS when a call is missed (the standout for clinics)
  • No-show follow-up
  • Long-term nurture + stale-lead re-engagement
  • Review request after a win
Gotcha - bulk sendsConsent + a dry-run/count check before any marketing blast. Never send to a live-queried list.
9

Google presence

Business Profile: business.google.com → create, pick category, service-area + cities, website/phone in their own fields, description in the description field (no URLs in the text), verify. Use a dedicated Google account per brand - never mix brands.

Search Console: add a Domain property → add the TXT at DNS → verify → submit sitemap.xml → URL-inspect key pages and Request Indexing.

Final verification - prove it works

  • Test message on web chat, IG, FB - bot replies in the right voice
  • Bot asks for name/email/phone, books, and the confirmation email + SMS arrive WITH the meeting link
  • Cold email appears in the system and the bot responds (if Phase 6 done)
  • Voice agent answers, routes, and books with details captured
  • Pricing/offer the bot gives is current (no stale snapshot content)
  • Confirmation workflow shows Published, not Draft

Replicating at scale

Build it once, then save a GHL Snapshot (workflows, calendars, custom fields, pipelines travel in snapshots; confirm whether the bot + knowledge base clone, or paste those per account from the saved files).

  • Collect each client's compliance docs early
  • Spin up their branded sub-account → load the snapshot
  • Swap branding + their offer specifics → attach their approved phone bundle
  • Submit each client's regulatory bundle at the start of the week so it's approved by handover
